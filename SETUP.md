## âš ï¸ Important Notes

### Demo Mode
- **Data Storage:** All data is stored in-memory (RAM)
- **Data Persistence:** Data is **LOST** when you restart the server
- **Production:** Configure Supabase for persistent storage

### Authentication
- Uses NextAuth with credentials provider
- LinkedIn OAuth is disabled (no credentials configured)
- Passwords are hashed with bcrypt

### AI Features
- Requires valid OpenAI API key
- Uses GPT-4o model
- Costs ~$0.01 per job description parsing

## ğŸ› Troubleshooting

### "Port 3000 is already in use"
```bash
# Kill the process on port 3000
lsof -ti:3000 | xargs kill -9

# Or use a different port
npm run dev -- -p 3001
```

### "OpenAI API key not configured"
- Check `.env.local` exists in project root
- Verify OPENAI_API_KEY line has your actual key (no spaces)
- Restart the dev server: Ctrl+C, then `npm run dev`

### "Module not found" errors
```bash
# Clear cache and reinstall
rm -rf node_modules .next
npm install
npm run dev
```

### "Invalid credentials" on login
- Make sure you created an account first at `/signup`
- Data is in-memory, so restart = data loss
- Try creating a fresh account

### Build errors
```bash
# Run type checking
npm run build

# This will show any TypeScript errors
```
